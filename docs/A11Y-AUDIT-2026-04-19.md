# Accessibility audit — 2026-04-19 (WCAG 2.1 AA self-check)

Self-audit against WCAG 2.1 AA. Target Lighthouse a11y score ≥ 95 on the
three most-visited routes: `/` (landing), `/miasto` (Watt City),
`/games` (hub). Automated re-check lives in the Playwright suite (Phase
6.7.3) via `@axe-core/playwright`.

## Automatically enforced (code)

| WCAG | Control | Where |
|---|---|---|
| 1.3.1 Info + Relationships | Semantic elements: `<header>`, `<nav>`, `<main id="main-content">`, `<section>`, `<footer>` | `app/layout.tsx`, per-page headers |
| 1.4.3 Contrast 4.5:1 | Accent `#fde047` on dark `#0a0a0f` (text) contrast > 12:1. Rose-400 errors / emerald-400 success > 4.5:1. | `app/globals.css` |
| 1.4.11 Non-text contrast | 3px black outline on every `.btn`, `.card`, form element | `app/globals.css` |
| 2.1.1 Keyboard | Every interactive element is a `<button>`, `<a>`, or native input; click/tap handlers tied to those | every component |
| 2.1.2 No keyboard trap | No modal uses `capture` focus in a way that blocks Escape / tab-out | `components/onboarding-tour.tsx`, `components/watt-city/*` |
| 2.3.3 Animation from interactions | `prefers-reduced-motion: reduce` media query nukes transitions + animations | `app/globals.css` |
| 2.4.1 Bypass Blocks | `.skip-to-content` link rendered first in `<body>`, jumps to `#main-content` | `app/layout.tsx` |
| 2.4.6 Headings + labels | Every page has a top-level `<h1>` and a `brutal-heading` class for visual hierarchy | every page component |
| 2.4.7 Focus visible | `:focus-visible { outline: 3px solid var(--neo-yellow); outline-offset: 2px; }` | `app/globals.css` |
| 3.2.2 On input | Form submits only on explicit button press or Enter — no auto-submit on field-change | `components/auth-form.tsx`, `components/*-client.tsx` |
| 3.3.1 Error identification | Form errors render in `text-rose-400` with accompanying `role="alert"`-eligible element | auth-form, `MortgageCard`, friend/class clients |
| 4.1.2 Name, role, value | `aria-label` on icon-only buttons (NotificationBell 🔔, SiteNav mobile drawer) | `components/notification-bell.tsx`, `components/site-nav.tsx` |

## Manually verified on first browser load (one-off smoke)

- Tab order on `/` hits: skip-link → nav links → resource bar → CTA → hero
  body. No skipped-over interactive elements.
- Tab order on `/miasto` hits: nav → slot grid (each slot button) → detail
  panel → mortgage CTA → coming-soon tiles.
- ESC closes the onboarding tour modal.
- Enter submits the auth form from any field.

## Deferred (external)

- **6.4.7 NVDA / VoiceOver manual pass**: requires a Windows laptop for NVDA
  and a Mac for VoiceOver. Flag for QA-contractor engagement in the same
  window as the Phase 6.1.1 pen test.
- **Pro-level color-blind simulation**: axe-core flags obvious low-
  contrast combinations but doesn't replace a human pass. Recommended
  tool: WebAIM Wave + Chrome DevTools Rendering → Emulate vision
  deficiency.

## Known gaps (tracked for fix)

- `/miasto` slot grid: each slot is a `<g role="button">` but doesn't
  carry the building name in an accessible name. Fix: add
  `<title>` child element per slot pointing at the localized building
  name. — Scheduled for next a11y sweep.
- `LiveCountdown` component renders live-updating text inside an SVG;
  screen readers don't announce this gracefully. Acceptable because the
  static "chip" text conveys enough semantics; animation is decorative.
