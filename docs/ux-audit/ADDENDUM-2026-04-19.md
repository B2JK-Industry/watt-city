## New findings surfaced by the re-test

1. **`/dla-szkol/materialy` curriculum counts show `Ekonomia 0 kodów`** while financial-literacy content is the whole product thesis. Either the curriculum map in `lib/curriculum.ts` really has 0 entries tagged `Ekonomia` (likely a tag-naming issue — codes are under `WOS 14` because in the Polish MEN, personal-finance sits under `WOS`), or the counter is broken. Either way the pitch PDF should not show `0` next to `Ekonomia`. Severity: HIGH for PKO demo. (Evidence: live fetch of `/dla-szkol/materialy`.)

2. **Nav discovery for /dla-szkol is the single biggest remaining miss.** Persona 3 (teacher) and Persona 5 (PKO PM) still cannot get to the landing page without knowing the URL. Adding the nav entry is a 15-minute change to `components/site-nav.tsx`.

3. **V4.8 Content Machine banner is position #1 on anonymous landing**, above the main hero. Visually that means the first thing an anonymous visitor reads is "this will be built in Q3 2026." That pushes the actual value proposition below the fold. Consider moving the banner to the side or below the top CTA row. (Evidence: `curl -o /tmp/landing-v2.html https://watt-city.vercel.app/` — "🚀 Content Machine" appears before "Wygeneruj Waty").

4. **Hero copy contradicts the V4 pivot**. The hero still includes `"pojedynek na kod"` even though `/duel` now 404s. An anonymous user who wants to try the "pojedynek" the hero advertises cannot. (Evidence: `/` hero text.)

5. **Leaderboard realism issue persists** — 3rd place on anonymous landing is `maxwell012` with `80 W$`. The audit test user `uxverify_04870a` would enter the top-5 with a single game completion. For a PKO pitch showing "gamified finance education" this signals "barely anyone plays." The D7 flag-ramp took care of UX feature-flag coverage but didn't add demo users. Recommendation: invoke `POST /api/admin/seed-demo-school` once at deploy and let the simulated activity fill the leaderboard. V4.4 endpoint exists.

6. **`city-scene.tsx` is the largest single source of remaining Slovak on live**. A single file touches all of: `Finančný kvíz` (:254), `Pamäťové páry` (:274), `Kurzový šprint` (:314), `AI výzva dňa` (:376, :1001, :1002, :1004, :1005). Fixing that one file removes every Slovak game name from the landing + `/games` night-scene. (Evidence: `grep -n 'Finančný\|výzva dňa\|Pamäťové\|Kurzový' components/city-scene.tsx`.)

7. **Under-16 consent UX still silent**. Registered `uxverify_04870a` with birthYear 2013. API returned `{needsConsent:true, pendingConsentToken:"…"}` but session is live, `/api/me` returns `authenticated:true`. No in-app "waiting for parent approval" card visible. Same concern as original report — unchanged.

8. **V4.10 SKIN=pko still untestable in prod without a redeploy**. D8 claims "propagates through the whole app" but the only runtime toggle is a build-time env var. A query-param override in non-prod envs would make this PM-checkable. (Design ref: `docs/pko-demo-mode.md` per D8 diff.)
