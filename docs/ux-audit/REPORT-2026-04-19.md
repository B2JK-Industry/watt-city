## Executive summary

The single most important finding this audit produces is a **branch-management catastrophe**: the V3 mechanics refactor and nine of the ten V4 classroom-pivot deliverables live on `origin/watt-city-v3v4`, which was never merged, and V4.6 on watt-city was **reverted** after landing (commit `11bc684 Revert "feat(v4):V4.6 …"`). The live site therefore ships Phase 1-5 + V2 refactor + the mechanics fix merge and nothing else. Everything the operator prompt and session summaries describe as "shipped" for V3/V4 is, for a user staring at a browser, **invisible**.

Concretely, the top 8 items you should stop reading after (and start fixing):

1. **`/dla-szkol` returns HTTP 404** — the entire V4 classroom landing page (teacher hero, value prop, "podstawa programowa" badge, PKO pitch surface) is not deployed. The PKO PM persona has nothing to land on. Code exists on `watt-city-v3v4`; not on live.
2. **`/duel` is still in the main nav bar** — V3.6 called for removal. Kid persona sees "Pojedynek" between "Gry" and "Liga" and the feature is a Slovak-titled orphan with no resources + no progression.
3. **Landing, dashboard, and every game listing are a Polish-Slovak-English pidgin** — "🛖 Drevená búda v Nikiszowci · Tier 1" is printed to every logged-in user, as are game titles "Finančný kvíz / Pamäťové páry / Matematický šprint / Kurzový šprint / Premiešané slová" while building labels and nav are Polish. This alone would lose a PKO demo in 10 seconds.
4. **V3.1 "city-first" identity was never finished** — `CityLevelCard` + Slovak-labelled `PlayerBuilding` both render on `/`, competing for the mental model. Nav badge is `"🛖 Drevená búda"` (Slovak tier title) instead of the design-called-for `"⚡ Level X · Grid: ±Y/h"`.
5. **`PostGameBreakdown` component is shipped but never mounted** — `components/post-game-breakdown.tsx` exists; zero imports under `app/games/**`. `v2_post_game_modal` flag is at 50% but no game client ever mounts the modal, so the flag controls nothing the user sees.
6. **V3.2 starter kit (50 coins + 50 bricks on register) is not implemented** — a fresh 12-yo persona starts with `watts:0, coins:0, bricks:0`. Sklepik unlock requires `earn-coins:50`; Małą elektrownia requires `earn-coins` gating. The 2-hour blind-grind cliff the session summary called out as CRITICAL F2 is still there.
7. **Three teacher/parent pages exist but are unreachable from the nav** — `/class`, `/parent`, `/pko` are in the codebase, respond 200, but appear in zero nav entries or CTA buttons. A teacher would need to type the URL. A parent who got no invitation cannot get to `/parent` by link.

PKO embarrassment risk if the pitch deck is shown tomorrow: **HIGH**. Five of eight items above are visible in the first five clicks a sceptical PM would make.
